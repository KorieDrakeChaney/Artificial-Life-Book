Conway's Game of Life

Conway’s Game of Life is a very simple but rewarding project, understanding it and doing it on your own is such a great feeling. I will be walking you through it with the basic logic while creating programming building blocks along the way.

We’re evolving from a one-dimensional CA to now a two-dimensional CA adding complexity as we will have a larger neighborhood of 9, contrast to the simple neighborhood of 3 in Elementary Cellular Automata.

This carries the same concept of a world of cells, which carry 2 states, 0 or 1, alive or dead.

As for the rules, there are 3 basic rules:

  • (1) Survival: If a cell is in state 1 (alive) and has 2 or 3 neighbors in state 1, then the cell survives, i.e., remains in state 1.

  • (2) Birth: If a cell is in state 0 and has exactly 3 neighbors in state 1, then in the next time step the cell goes to state 1.

  • (3) Death: A cell in state 1 dies (goes to state 0) of loneliness if it has 0 or 1 neighbors. Also, it dies because of overcrowding if it has 4 or more neighbors.


With these simple rules, many patterns can occur. Common pattern types include still lifes, which do not change, oscillators, which return to their initial state after several generations, and spaceships, which travel across the grid !
